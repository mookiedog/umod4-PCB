# Production: Getting a Board to JLCPCB

## JLCPCB Parts: Basic vs Extended

JLCPCB's parts catalogue is at https://jlcpcb.com/parts — a subset of LCSC.com
available for their assembly service. Results show whether each part is Basic or
Extended. There is a filter checkbox to restrict to Basic parts only.

Basic parts carry no added setup fee. Extended parts incur a small fee per unique
part number. Prefer Basic where a suitable equivalent exists.

---

## Build directory

All generated files go into `build/`, which is gitignored. It has two subdirectories:

| Directory | Contents | Owner |
|-----------|----------|-------|
| `build/cam/` | Gerbers and drill file generated by the CAM processor | Eagle CAM |
| `build/jlcpcb/` | Everything that will get uploaded to JLCPCB, plus the HTML BOM | Script + ULP |

To start completely clean, delete `build/cam/` before running the CAM processor.
Eagle will recreate it. The script always regenerates `build/jlcpcb/` from scratch.

---

## Step 1 — Pre-flight Checks in Eagle

### Ratsnest

In the schematic editor: verify that there are no air wires.
A 'ratsnest' command must return "Nothing to do!"

### ERC

In the schematic editor: run the ERC. It should show only known/approved errors and
warnings.

### DRC

In the board editor: run the DRC. It should show only known/approved errors and
warnings.

The DRC only checks **visible** layers. Before running, make exactly these layers
visible and turn everything else off:

- Top, Bottom
- Pads, Vias, Unrouted
- Dimension
- tOrigins, bOrigins
- tNames, bNames
- tStop, bStop
- tKeepout, bKeepout
- tRestrict, bRestrict
- Drills, Holes, Milling

In particular make sure **tPlace** and **bPlace** are **off**. If they are on, they will
produce spurious "Stop Mask" errors from package outline geometry.

---

## Step 2 — Generate Gerbers (CAM Processor)

Open the CAM processor in Eagle and load the project's `.cam` file
(e.g. `jlcpcb_2_layer_v72.cam`). Click **Process Job**.

Warnings about polygons causing large plot data are harmless — click **Yes to All**
and continue. JLCPCB has no problem with the resulting files.

The CAM job writes 12 files to `build/cam/`:

| File | Contents |
|------|----------|
| `<name>.GTL` | Top Copper |
| `<name>.GBL` | Bottom Copper |
| `<name>.GTO` | Top Silkscreen |
| `<name>.GBO` | Bottom Silkscreen |
| `<name>.GTP` | Top Solder Paste |
| `<name>.GBP` | Bottom Solder Paste |
| `<name>.GTS` | Top Solder Mask |
| `<name>.GBS` | Bottom Solder Mask |
| `<name>.GKO` | Board Outline / Slots |
| `<name>.XLN` | Drill Data (Excellon) |
| `<name>.gpi` | General Info (not sent to JLCPCB) |
| `<name>.dri` | Drill Info (not sent to JLCPCB) |

JLCPCB requires only the ten files with **uppercase** extensions.

## Step 3 — Run the Pre-Production Script

From the project root in WSL:

```
python3 preproduction.py
```

The script:

- **Checks** that all ten gerber/drill files in `build/cam/` are newer than both
  the schematic and board files. Reports stale files and tells you to re-run the
  CAM processor if needed.
- **Generates** `build/jlcpcb/<name>-bom.html` — a human-readable BOM with two
  sections: JLCPCB assembly parts and DNP / hand-solder / other parts. LCSC part
  numbers are clickable links to the JLCPCB component search page so you can verify
  availability and Basic vs Extended status before ordering.
- **Generates** `build/jlcpcb/bom.csv` — the BOM in JLCPCB upload format.
- **Generates** `build/jlcpcb/<name>_top_cpl.csv` — the centroid file, by invoking
  Eagle in batch mode with the modified `jlcpcb_smta_exporter_v7.ulp`.
- **Creates** `build/jlcpcb/<name>.zip` — the ten gerber/drill files ready for
  JLCPCB upload. Only created if the gerber check passes.

A clean run looks like:

```
============================================================
 Pre-Production Check: <name>  (2026-07-02 10:30)
============================================================
 [PASS] Gerbers are current
 [PASS] BOM: 22 PCBA parts, 14 other/DNP
 [PASS] CSV:  build/jlcpcb/bom.csv
 [PASS] HTML: build/jlcpcb/<name>-bom.html
 [PASS] CPL:  build/jlcpcb/<name>_top_cpl.csv
 [PASS] ZIP:  build/jlcpcb/<name>.zip (52 KB)
============================================================
```

Exit code is 0 on a clean run, 1 if anything requires attention.

**Note:** Every output file is overwritten each run. `build/jlcpcb/` is fully
reproducible — there is no user-authored content there that can be lost.

---

## Step 4 — Review the HTML BOM

Open `build/jlcpcb/<name>-bom.html` in a browser. Walk through the JLCPCB Assembly
section and click each part number link to confirm:

- The part is in stock at JLCPCB
- Basic vs Extended status matches expectations
- The part number still refers to the correct component

---

## Step 5 — Upload to JLCPCB and Review

Go to https://www.jlcpcb.com. Log in or create an account.

Click **Instant Quote**. Drag and drop `build/jlcpcb/<name>.zip` onto the Gerber
upload area and wait for JLCPCB to process the file.

Verify the board parameters match your design intent. Typical 2-layer settings:

| Parameter | Value |
|-----------|-------|
| Base material | FR4 |
| Layers | 2 |
| Different Design | 1 |
| Delivery format | Single PCB |
| Thickness | 1.6 mm |
| PCB Color | Green |
| Silkscreen | White |
| Surface finish | HASL (with lead) |
| Copper weight | 1 oz |
| Via covering | Tented |
| Min hole size | 0.3 mm |
| Board Outline Tolerance | Regular |
| Confirm Production File | Yes |
| Mark on PCB | Order number (specify position) |
| Electrical Test | Flying probe fully test |
| Gold fingers | No |
| Castellated holes | No |
| Edge plating | No |

Enable the PCB assembly option and upload:
- `build/jlcpcb/bom.csv` — the BOM file
- `build/jlcpcb/<name>_top_cpl.csv` — the centroid file

If the preview or BOM review reveals a problem, fix it in Eagle, increment the
silkscreen version letter, and repeat from Step 2 (CAM).

## Step 6 — Place the Order and Tag the Release

Once you are satisfied with the JLCPCB preview and ready to place the order, tag
the current source state and create a GitHub Release to permanently record what
was submitted.

```bash
git tag -a 4v2c -m "First order — brief description of what changed from previous"
git push origin 4v2c
gh release create 4v2c build/jlcpcb/* \
  --title "umod4 4v2c" \
  --notes "First order. Fixed R3 designator position from 4v2b."
```

The `build/jlcpcb/*` glob attaches the zip, bom.csv, cpl.csv, and HTML BOM as
release assets — a permanent record of exactly what was sent to JLCPCB.

Then place the order.

## Parts Placement Review

After uploading the BOM and CPL, JLCPCB shows a placement viewer. In practice this
viewer frequently shows parts offset or with incorrect rotations — do not be alarmed.
Check the box to **verify parts placement before production** and proceed through the
purchase flow.

About an hour after you confirm the PCB order a JLCPCB engineer will manually review
the placement. They will send two images: the original placement as extracted from your
files, and their corrected version. Carefully verify their corrections match your
intent — in particular check that polarised parts (LEDs, diodes, electrolytics) have
the correct orientation.

JLCPCB engineers are generally reliable at catching and correcting placement issues.
If something is wrong with the PCB itself they will cancel the job and contact you
before proceeding.
